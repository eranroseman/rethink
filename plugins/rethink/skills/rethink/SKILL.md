# Rethink

Apply the Rethink directive — active this session via SessionStart — to this
task in full, regardless of whether it looks tactical.

In short: requirements first, then research how this class of problem is solved
well elsewhere (cite sources), then design the clean-slate answer from those —
treating the current implementation as gap-measurement, not as the template.
End with trade-offs and a migration sketch.
